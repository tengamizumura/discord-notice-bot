# Discord Notice Bot (24時間稼働の自動通知システム)

特定のDiscordチャンネルでの発言を検知し、指定したユーザーへ自動でDM通知を送るBotです。
無料のPaaS環境（Render）と外部監視ツール（UptimeRobot）を使用し、24時間稼働し続けています。

## 主な特徴と工夫した点

1. **インフラの永続化**
   - Renderの無料枠（15分でスリープ）によるBotの停止を防ぐため、`Flask`を用いたダミーのWebサーバー（`keep_alive.py`）をバックグラウンドで起動。
   - 外部サービス（UptimeRobot）から5分間隔でHTTPリクエストを送信させることで、サーバーを常に起こし続けるアーキテクチャを採用しています。
2. **セキュアな設計と型変換**
   - DiscordのTOKENだけでなく、監視対象の「チャンネルID」もハードコーディングを避け、Renderの環境変数（Environment Variables）から読み込む設計にしています。
   - 環境変数は文字列(`String`)として取得されるため、DiscordのID判定を正確に行うために `int()` で明示的な型変換を行っています。
3. **CI/CD（自動デプロイ）の導入**
   - GitHubリポジトリの更新（Push）をRenderが検知し、自動で最新のコードがサーバーにデプロイされる仕組みを構築しました。

## 使用技術

- **言語:** Python 3.x
- **ライブラリ:** `discord.py`, `Flask`
- **インフラ・監視:** Render (PaaS), UptimeRobot
- **バージョン管理:** Git, GitHub
- **開発環境:** WSL2 (Ubuntu), VS Code

## 動作の仕組み

1. 監視対象のチャンネル（IDを環境変数で複数指定）で新しいメッセージが投稿される。
2. `discord.py` のイベントリスナーがそれを検知。
3. 発言者の名前とメッセージ内容を整形し、指定したユーザーのDMへ即座に転送。
